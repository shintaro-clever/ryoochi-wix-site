# Startup Manual Scope

## 対象読者
- 非エンジニアの案件担当者
- Wix Studio で案件立ち上げを進める運用担当者

## 目的
- 案件開始時に何を準備し、何を確認し、誰に依頼するかを整理する
- 技術詳細に踏み込みすぎず、立ち上げ判断を進めやすくする

## 対象範囲
- 案件開始前に確認すべき前提条件
- 必要な素材、文言、画像、アクセス権の確認
- Wix Studio 側で進める作業の入口整理
- 管理者や技術担当へ確認すべきタイミングの判断

## 対象外
- Git の詳細操作
- CLI の詳細手順
- コード修正や実装フローの説明
- 開発環境構築の深いトラブルシュート

## 管理者へ確認すべきケース
- Wix 接続や権限周りで判断できないとき
- CLI や Git 操作が必要になったとき
- Repo 内の技術文書だけでは判断できないとき
- 本番影響が出る変更や構造変更の相談が必要なとき

## 運用上の考え方
- 立ち上げマニュアルは非エンジニア向け運用文書とする
- 技術的な深掘りは対象外とし、必要な場合は管理者確認へ切り替える
- 技術手順書は `docs/wix/` などの別文書で管理し、この文書と混ぜない
