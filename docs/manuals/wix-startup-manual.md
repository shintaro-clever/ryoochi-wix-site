# Wix Startup Manual

## このマニュアルについて
- この文書は、非エンジニアが Wix Studio 案件を立ち上げるときに最初に読むためのものです
- 目的は、案件開始時にやることの順番を迷わないようにすることです
- 技術的に深い操作は扱わず、必要な場合は管理者へ確認します

## 1. 何を準備するか

### 先に揃えるもの
- 案件名
- Wix Studio の対象案件または新規案件の置き場
- 使用予定の文言、画像、ロゴ
- 誰が最終調整を担当するかの確認

### あらかじめ決めておくこと
- まず検証する対象は「本番全体」ではなく、1ページまたは1セクション相当の最小単位にする
- 最終的な見た目の仕上げは Wix Studio 側で行う
- 日常更新はできるだけ Wix Studio 上で完結させる

## 2. 何を確認するか

### 案件開始時に確認すること
- Wix Studio に入れる状態か
- 必要な画像や文言が最低限そろっているか
- まず何を検証するかが決まっているか
- 本制作ではなく、最小検証から始める前提が共有されているか

### 作業に入る前の確認
- 静的原型が用意されているか
- 誰が Wix Studio 上で最終調整するか決まっているか
- 自分で進めてよい範囲と、管理者へ確認すべき範囲が分かっているか

## 3. どこまで自分でやるか

### 自分で進めてよいこと
- 案件名、画像、文言などの準備
- Wix Studio 上でのページ確認
- 画像差し替え、文言差し替え、配置確認などの通常調整
- 検証対象が最小単位に収まっているかの確認

### 自分だけで進めないこと
- 技術手順が必要な接続作業
- Git や CLI の操作判断
- 本番ページ全体へ一気に広げる判断
- 構造を大きく変える判断

## 4. 困ったときどうするか

### よくある詰まり方
- Wix Studio に入れるが、どこから始めればよいか分からない
- 画像や文言が足りず、検証対象が決められない
- 本番全体を先に作りたくなってしまう
- 画面上の調整で済むのか、管理者確認が必要なのか判断できない

### 迷ったときの考え方
- まずは最小単位で検証する
- 通常の見た目調整なら Wix Studio 側で進める
- 技術的な判断が必要なら、その場で止めて管理者へ確認する

## 5. 管理者に確認する場面
- 接続や権限で止まったとき
- Git や CLI の操作が必要と言われたとき
- 静的原型を作り直す必要がありそうなとき
- Wix Studio 上の調整だけでは吸収できない構造変更が出たとき
- 本制作へ進むか、方針を修正するかの判断が必要なとき

## 6. 最後に見るポイント
- 今やっているのが「最小検証」か「本制作」かを混同しない
- 仕上げは Wix Studio 側で行う前提を保つ
- 技術的な深掘りは自分で抱え込まず、管理者確認へ回す

## 補足
- Git や CLI の詳細はこのマニュアルの主目的ではありません
- 技術手順が必要な場合は `docs/wix/` 配下の文書を参照するか、管理者へ確認してください
