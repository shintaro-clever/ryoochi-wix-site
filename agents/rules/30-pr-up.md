# 30-pr-up

目的:
- `node scripts/pr-up.js` を唯一入口とする運用を明確化する。
- PR更新手順の参照点を一元化する。

---

## PR作成のタイミング

**ユーザーが「PRあげてください」「PRを作成してください」と明示した場合のみ実行する。**

作業途中・コミット後であっても、明示的な指示がない限り push・PR作成は行わない。

---

## 実行手順

必ずこの順で実行する：

```bash
unset GITHUB_TOKEN
node scripts/pr-up.js
```

---

## 禁止事項

以下は `pr-up.js` が失敗した場合も含め、いかなる理由でも使用禁止：

- `gh pr create`
- `gh api -X POST .../pulls`
- 手動 `git push` + PR作成
- その他 `pr-up.js` を経由しない方法

---

## 失敗時の対応

`pr-up.js` が失敗した場合：

1. 失敗コマンドとエラー出力をそのまま報告して停止
2. 代替手段を試みない
3. ユーザーの指示を待つ

---

## 競合解消後の報告

PR 競合を解消した後の報告は簡略化しない。少なくとも以下を必ず含める：

1. 何の競合だったか
2. どの変更同士を両立したか
3. どのファイルで何を残したか
4. テスト再実行の有無
5. PR URL と merge commit

`競合を解消しました` だけで終わらせない。

---

## コミットメッセージ要件

PR に含まれる各コミットは **件名 + 本文** を必ず持つこと。

本文に含める内容：
1. 何を変えたか（対象ファイル・関数名）
2. なぜ変えたか（背景・Issue番号）
3. 影響範囲（破壊的変更の有無）
4. テスト結果（`npm test` 通過 / スキップ理由）

本文なし・件名だけのコミットは規則違反。`pr-up.js` 実行前に必ず確認すること。
